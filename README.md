# Автоматизация сверки накладных для аптеки

## О проекте

Этот проект автоматизирует сверку документов между входящими выгрузками и аптечными учетными файлами.

Ноутбук обрабатывает:
- входящие файлы из папки `incoming_files/`;
- аптечные CSV-файлы из папки `pharmacy_files/csv/correct/`;
- формирует итоговый результат с дополнительными полями для сверки.

## Структура папки

```text
pharmacy-invoice-reconciliation-automation/
├── README.md
├── pharmacy_invoice_reconciliation.ipynb
├── incoming_files/
│   ├── Входящие 01.csv
│   ├── Входящие 01.xlsx
│   ├── Входящие 02.csv
│   ├── Входящие 02.xlsx
│   ├── Входящие 03.csv
│   └── Входящие 03.xlsx
└── pharmacy_files/
    └── csv/
        └── correct/
            ├── 366.csv
            └── А123.csv
```

## Что делает ноутбук

- читает входящие CSV/XLSX-выгрузки;
- читает аптечные учетные CSV-файлы;
- сопоставляет записи по бизнес-логике;
- добавляет поля для проверки накладной, суммы и даты;
- сохраняет итоговые результаты в папку `output/текущая_дата`.

## Какие навыки показывает проект

- работа с несколькими источниками данных;
- очистка и сопоставление записей;
- автоматизация отчетности;
- обработка Excel- и CSV-файлов;
- решение прикладной бизнес-задачи.
