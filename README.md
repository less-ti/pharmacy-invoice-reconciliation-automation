# Автоматизация сверки накладных для аптеки

## О проекте

Этот проект посвящен автоматизации сверки документов между несколькими источниками данных.  
По сути, это прикладной кейс по оптимизации рутинного процесса, где вместо ручной сверки большого количества табличных файлов используется воспроизводимый и структурированный пайплайн обработки данных.

Проект особенно ценен тем, что он ближе к реальной бизнес-задаче, чем классические учебные датасеты: здесь важно не просто построить график или посчитать агрегат, а надежно сопоставить записи, учесть различия в формате входных файлов и получить итоговый результат, удобный для дальнейшей работы.

## Что делает проект

Ноутбук:
- читает входящие выгрузки из папки `incoming_files/`;
- читает аптечные CSV-файлы из папки `pharmacy_files/csv/correct/`;
- подготавливает данные к сопоставлению;
- выполняет сверку записей по логике проекта;
- формирует итоговый файл с результатами проверки;
- сохраняет результат в папку `output/текущая_дата`.

## Что сделано в проекте

- организована работа с несколькими входными файлами;
- показан подход к стандартизации и сопоставлению данных;
- реализована автоматизация ручного процесса сверки;
- сформирован итоговый выходной файл для последующей проверки.

## Почему проект важен

Этот кейс показывает сильную прикладную сторону аналитики:
- умение работать с реальными файлами, а не только с идеально подготовленными датасетами;
- умение автоматизировать процесс, который в ручном режиме занимает время;
- внимательность к структуре данных и логике сопоставления;
- способность превращать хаотичные выгрузки в понятный итоговый результат.

## Какие навыки показывает проект

- работа с несколькими источниками данных;
- очистка и сопоставление записей;
- автоматизация отчетности;
- обработка Excel- и CSV-файлов;
- прикладная бизнес-аналитика;
- структурирование рутинного процесса через Python.

## Результат

Итогом работы является воспроизводимый ноутбук, который можно использовать как основу для регулярной сверки документов и подготовки отчетного результата без ручной обработки каждой выгрузки.
